Best practices

Don’t concatenate strings

// Bad

Hey {name}, welcome back!

// Good

Hey {name}, welcome back!

Example: “red pencil” in English → “crayon rouge” in French (note the reversed order of words). Put punctuation inside localization strings Different languages may use punctuation differently (for example, different spacing) or even different characters. // Bad

Username:

// Good

Username:

Example: “Username:” in English → “Nom d’utilisateur :” in French (note the space before the colon).

Don’t reuse translations in different contexts The same English strings maybe translated differently in different contexts.

// Bad

Name

// Good

Name Example: “Bookmark” (heading) and “Bookmark” (button) in English → “Закладка” and “Добавить в закладки” in Russian. ( {props.resultCount} results ) : ( {props.resultCount} result ) // Good // TODO: we don't have a proper solution yet Example: “1 dog, 2 dogs, 5 dogs” in English → “1 собака, 2 собаки, 5 собак” in Russian. Escape special characters Characters like & or ’ should be escaped using HTML entities: // Bad Name can’t be empty

// Good

Name can’t be empty Other characters: Single quotes: ‘ ’ → ‘ ’ Double quotes: “ ” → “ ” Em (long) dash: — → — Ampersand: & → &

Don’t use dangerouslySetInnerHTML or react_lnrs Send data from the backend that doesn’t require using dangerouslySetInnerHTML on the frontend, ideally translate on the frontend:

// Bad import lnrs from 'react_lnrs';

// Bad

// Good

Thanks for sharing! You’ve been entered to win a {this.props.sweepstakesAmount} shopping spree! Don’t use wf.translate All React code should use the Lnrs component:

// Bad

<TextInput label={wf.translate('Email', 'Email')} />

// Good

<TextInput label={Email} />
